Cap 18 - Repositorio local Git e Repositório remoto GitHub

Aula 1: Oque é Git e Github?

Repositorio local / remoto:
Lugar aonde é armazenado arquivos

Git:
Versionamento e repositório LOCAL automatico de arquivos, gerenciador de versões, se for necessario voltar alguma versão é possível.

Não é possivel acessar esses arquivos de outro computador.

Se o HD do computador apresentar problemas você perde tudo 


GitHub:
Repositório REMOTO, sendo possivel acessar os arquivos de outro computador


É interessante utilizar as duas ferramentas em conjunto, ou seja, versionar todo o seu projeto e inserir esse repositório local Git no repositório remoto GitHub.

Para deletar um repositorio primeiro apaga o local e depois tem que apagar o remoto

Aula 10: GitHub em vários PCs

Não estou em casa e quero usar o git github em outro computador

Navegador / vscode / git / github desktop 

procedimento

1. abrir github desktop e logar

2. file > options > git > save

3. create a tutorial repository > selecionar o repositorio > clone > local para salvar o clone > open vscode > editar e realizar todas os trabalhos > commit > push origin

4. Sair de tudo > deixar somente github desktop > repositorio > remove e selecionar o checkbox de enviar para lixeira > file > sign out > apagar os dados do git > esvaziar lixeira


ir para outro computador, trabalhar, jogar as alterações na nuvem, e ir embora apagando todos os registros desse outro computador.
